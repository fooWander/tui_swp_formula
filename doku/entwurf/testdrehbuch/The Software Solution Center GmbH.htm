Testdrehbuch

Nur ein strukturiertes Testen gewährleistet, dass sämtliche Abläufe korrekt durchgespielt und allfällige Fehler gefunden werden.

Ist die Entscheidung für eine Software gefallen, die Daten testhalber migriert, geht's ans Testen. Dies ist eine unbeliebte Arbeit, aber extrem wichtig! Testen sie lieber etwas mehr als zu wenig.

Oftmals wird einfach so "drauflos" getestet. Ein Testdrehbuch hilft, Ordnung in die Testerei zu bringen. Man sollte systematisch vorgehen. Vom einfachen Fall bis hin zu den komplizierten Fällen im Unternehmen. Es sollte immer mit Daten getestet werden, welche auch realistisch sind. Dabei macht es Sinn, alte Fälle des Betriebes nachzuspielen. Nur so kann man sicher sein, dass wirklich alles abgedeckt ist.

Wichtig ist auch, dass am Anfang immer mit denselben Zahlen und Eingaben getestet wird, bei denen man auch das Resultat genau kennt. Erst wenn diese Tests abgeschlossen sind, dann sollten "unmögliche" Sachen getestet werden. Damit schaut man, wie sich das System verhält. Kann das System Fehler erkennen? Lässt es zu, dass Fehleingaben gemacht werden?

Das Testdrehbuch ist Gold wert, sie werden es sehen. Sparen sie nicht am falschen Ort!
